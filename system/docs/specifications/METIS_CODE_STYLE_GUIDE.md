# METIS CODE STYLE GUIDE
Version: 1.0

Defines code style expectations.

## PHP

- use clear class and method names
- keep request handlers thin
- keep services focused
- avoid duplicate helpers
- prefer readable structure over clever shortcuts

## JavaScript

- use modular files
- use core services
- no inline scripts
- no random utility sprawl

## Naming

- services → PascalCase
- actions → `module.action`
- DB tables/columns → snake_case
